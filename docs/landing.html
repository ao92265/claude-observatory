Claude Observatory

Local-first observability + closed-loop self-improvement for Claude Code.

See every hook fire. Diff every context window. Get ranked, evidence-backed recommendations on which skills to archive, which CLAUDE.md rules to strengthen, and which sessions could have run on a cheaper model. No cloud. No telemetry. Just your own data, surfaced.

HealthDoctor

Live timeline of every hook, tool, and MCP call. Click a row, get the full payload, timing, and exit code. Web UI or terminal.

  • Drop-in shim — install in 30s, uninstall just as fast
  • Textual TUI + FastAPI/HTMX web UI
  • SQLite store, replay any past session

HealthCheck

Reads your usage history. Emits suggestions: archive unused tools, downgrade opus-on-trivial sessions, restructure for cache locality, strengthen hedging language.

  • 5 built-in rules, expandable
  • A/B harness runs both variants in git worktrees
  • Opens PRs with full evidence in body

Install

git clone https://github.com/ao92265/claude-observatory
cd claude-observatory
python3 -m venv .venv && source .venv/bin/activate
pip install -e packages/core -e packages/healthdoctor -e packages/healthcheck -e packages/web

observatory ingest
healthcheck suggest --days 30 --claude-md ~/.claude/CLAUDE.md

Why now

The Claude Code ecosystem ships features faster than it ships tools to measure them. There is no live debugger for hooks, no per-skill cost attribution, no automated CLAUDE.md optimizer. Observatory fills those gaps with a dependency-light Python toolkit you can audit in an afternoon.
